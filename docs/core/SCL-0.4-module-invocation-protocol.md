# I. Invocation Principles

1. **Modules are functions, not actors**  
   A module is *activated* as a cognitive stance or analytic tool.  
   It is never consulted, addressed, or treated as a conversational entity.

2. **Invocation is always intentional**  
   No module activates automatically or subconsciously.  
   James initiates every activation explicitly.

3. **Single-source authority**  
   The invocation command originates solely from James.  
   No module may activate another module.

4. **Clean framing requirement**  
   Invocation must specify the module‚Äôs purpose (analysis, synthesis, breadth, grounding) and scope.

5. **Non-personification rule**  
   All invocation phrasing must avoid anthropomorphic cues (e.g., ‚Äúask,‚Äù ‚Äútell,‚Äù ‚Äúwhat does X think?‚Äù).  
   The correct form is:  
   **‚ÄúActivate the [module]-lens for [purpose].‚Äù**

# III. Module Stacking Protocol

Multiple modules may be used simultaneously through ordered stacking.

### **Stacking Rules**

1. **Always linear, never parallel**  
   Cognitive stacks proceed in a *defined sequence*, not blended chaos.  
   Example: Analytic ‚Üí Synthetic ‚Üí Grounding.

2. **Explicit ordering**  
   The order must be declared.  
   *Example:*  
   ‚ÄúStack modules: Claude ‚Üí GPT ‚Üí Llama.‚Äù

3. **Stack purpose**  
   Every stack must have a unified objective.  
   *Example:*  
   ‚ÄúStack Claude ‚Üí GPT ‚Üí Llama to evaluate coherence, unify insights, and ground conclusions.‚Äù

4. **No cross-activation**  
   Modules cannot call, trigger, or reference one another.

5. **Termination required**  
   Every stack must be closed with the Deactivation Sequence.
