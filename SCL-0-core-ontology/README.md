# SCL-0 — Core Ontology Layer

**Layer Class:** Foundational  
**Volatility Expectation:** Lowest  
**Revision Frequency:** Rare and deliberate  
**Purpose:** Define SNAP’s irreducible conceptual anchors

---

## I. Role of SCL-0

SCL-0 contains the slowest-moving structural definitions within the SNAP Council Ledger.

This layer defines:

- Core evaluative axes
- Foundational governance concepts
- Non-procedural primitives
- Ontological commitments

These are not operational rules.
They are not case analyses.
They are not advisory outputs.

They are the conceptual bedrock upon which all other layers depend.

---

## II. Layer Hierarchy Context

SNAP layering follows structural dependency:

- **SCL-0:** Core Ontology  
- **SCL-1:** Governance DNA (mechanisms and protocols)  
- **SCL-2:** Operating Layer (execution patterns)  
- **SCL-3:** Ecosystem Evaluation (external systems and cases)

Changes in SCL-0 propagate downward.
Lower layers must not implicitly modify SCL-0 definitions.

---

## III. Modification Discipline

Revisions to SCL-0 must be:

- Explicitly versioned
- Structurally justified
- Consequence-aware
- Logged immutably

SCL-0 is not aspirational.
It is descriptive and consequence-grounded.

Ontology must move slower than mechanism.

---

## IV. Current Ontology Artifacts

- **SCL-0.1 — Sovereignty**
- **SCL-0.3 — Sustainability Quotient (SQ)**

Additional core primitives may be added only when necessary to preserve structural clarity across layers.  

---

## V. Stability Principle

If a concept:

- Defines how SNAP evaluates reality rather than how it operates within it,
- Anchors interpretation across multiple layers,
- Must remain stable across case evolution,

it belongs in SCL-0.

If it describes procedure or protocol, it belongs elsewhere.
